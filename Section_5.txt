AWS Certified Cloud Practitioner (CLF-C02)
Chapter 5: Content Delivery and Networking – Exam Notes
=======================================================

1. Amazon VPC (Virtual Private Cloud)
-------------------------------------
- AWS üzerinde kendi sanal özel ağını oluşturmanı sağlar.
- Bileşenler:
  * Subnets: Ağ bölmeleri (Public / Private)
  * Internet Gateway (IGW): VPC’yi internete bağlar
  * NAT Gateway: Private subnet → internet çıkışı (inbound yok)
  * Route Tables: Trafik yönlendirme kuralları
  * Security Groups (SG): Instance seviyesinde güvenlik (stateful)
  * Network ACL (NACL): Subnet seviyesinde güvenlik (stateless)

Notlar:
- SG = stateful, instance seviyesinde
- NACL = stateless, subnet seviyesinde
- IGW public subnet için, NAT Gateway private subnet için kullanılır

2. Amazon Route 53 (DNS & Routing)
----------------------------------
- AWS’in DNS ve domain kayıt servisi.
- Kullanım alanları:
  * Domain Registration (alan adı kaydı)
  * DNS Resolution (isim → IP çözümleme)
  * Routing Policies:
    - Simple Routing → tek kaynak
    - Weighted Routing → %’lere göre dağıtım (örn. %70 / %30)
    - Latency-based Routing → en düşük gecikmeli region’a yönlendirme
    - Failover Routing → yedek kaynağa geçiş
    - Geolocation Routing → kullanıcı lokasyonuna göre yönlendirme
    - Geoproximity Routing → coğrafi yakınlığa göre yönlendirme

Not: Route 53 sadece DNS değil, routing + health check de yapar.

3. Elastic Load Balancing (ELB)
-------------------------------
- Trafiği birden fazla EC2, container veya Lambda’ya dağıtır.
- Türleri:
  * ALB (Application Load Balancer): Layer 7, HTTP/HTTPS
  * NLB (Network Load Balancer): Layer 4, TCP/UDP, yüksek performans
  * GWLB (Gateway Load Balancer): Güvenlik cihazları için

Notlar:
- HTTP/HTTPS → ALB
- TCP/UDP → NLB
- Güvenlik cihazları → GWLB

4. Amazon CloudFront (CDN)
--------------------------
- AWS’in içerik dağıtım ağı (CDN).
- Kaynak: S3, EC2, ELB olabilir.
- İçerik, Edge Locations üzerinden cache edilip son kullanıcıya en yakın noktadan sunulur.
- Avantajlar:
  * Düşük gecikme (low latency)
  * Yüksek hız (high transfer speed)
  * AWS Shield & WAF entegrasyonu

Not: CloudFront = Edge Locations + CDN

5. Direct Connect & VPN
-----------------------
- VPN: On-premises ↔ AWS bağlantısı, internet üzerinden
  * Daha ucuz, hızlı kurulum
  * Internet hızına bağımlı
- Direct Connect: On-premises ↔ AWS özel fiber hattı
  * Daha güvenli, düşük latency
  * Yüksek bant genişliği ama daha pahalı

Notlar:
- Hızlı, ucuz → VPN
- Güvenli, yüksek performans → Direct Connect

Önemli Sınav Stratejisi – Chapter 5
-----------------------------------
- VPC: SG vs NACL farklarını iyi bil
- Route 53: Routing policy türlerini ayırt et (“Cost-based” yok!)
- ELB: ALB vs NLB farkını mutlaka bil
- CloudFront: CDN + Edge Locations
- Direct Connect vs VPN: Özel fiber vs internet farkı
